# 📖 Skill 指南：Java 注释导出为 Markdown

## 1. 何时使用
当用户提供一个或多个Java源代码文件（.java），并要求将其整理成文档、API接口说明书、或是代码学习笔记时，触发此Skill。

## 2. 核心工作流
1. **分析文件**：读取用户提供的 `.java` 源码文件路径或项目、模块、包内的全部 Java 文件。
2. **提取注释**：提取类上的 `@Deprecated`, `@since` 及所有 Javadoc 标准注释（`/** ... */`），以及核心的方法签名和内部逻辑注释（`//` 或 `/* ... */`）。
3. **格式化输出**：按模块、类、方法级别归类，自动生成规范的 Markdown 语法。
4. **验证结果**：检查导出的 Markdown 中是否包含无效的转义符，确保逻辑和层级清晰。

## 3. 约束与规则
- **注释提取**：必须完整捕获方法的返回类型、参数列表及对应的注释描述。文档中应包含方法的访问修饰符、泛型信息、异常抛出声明等。
- **完整性**：确保每个类、接口、枚举及其方法的注释都被提取，避免遗漏。若某些方法缺少注释，应在文档中标记为“未提供注释”。文档需要列出目标文件的方法、字段、构造函数等所有成员信息及其注释，确保文档完整性。
- **排版要求**：必须使用 Markdown 标题（#，##，###），方法需使用代码块（```java）包裹。
- **异常兜底**：如果遇到非法的Java语法导致解析失败，请提取原始代码文本，并在Markdown中用文本形式提示用户手动补充说明。
- **文件结构**：生成的Markdown文档的命名方式为`<package>.<ClassName>.md`，保存路径为euonia-portal项目下的 `docs/java/<module>/` 文件夹中，若不存在则自动创建。
- **API汇总**：每个模块包含一个apis.md文件，汇总该模块下所有类的API文档，并提供跳转链接，按包结构组织，确保用户可以快速定位到具体类的文档。
- **多文件处理**：若用户提供的是一个目录或模块，需递归扫描所有 `.java` 文件，并按包结构生成对应的Markdown文档。
- **头部信息**：每个生成的Markdown文件需包含模块名、包名、类名、作者信息等元数据，确保文档完整性。
- **方法签名解析**：在提取方法签名时，需正确处理泛型、重载、可变参数等复杂情况，确保生成的文档准确反映源码结构。
- **注解处理**：对于类和方法上的注解（如 `@Request`, `@Enqueue` 等），需在文档中以表格形式列出注解的元素及其默认值，确保用户理解注解的用途和配置方式。
- **示例代码**：在文档中提供方法的示例调用代码块，帮助用户快速理解方法的使用方式。
- **跨模块引用**：若方法参数或返回类型涉及其他模块的类，需在文档中提供跨模块的跳转链接，方便用户查看相关类的详细信息。
- **继承关系**：在类的文档中，需列出其继承的父类和实现的接口，并提供跳转链接，帮助用户理解类的层次结构。
- **异常处理**：在方法的文档中，需列出可能抛出的异常类型及其说明，帮助用户在调用方法时做好异常处理。
- **排除内部类和测试类**：默认情况下，内部类和测试类的文档不生成，除非用户明确指定需要生成内部类或测试类的文档。
- **文档更新**：在生成新的Markdown文档时，需检查是否已有同名文档存在，若存在则提示用户是否覆盖或保留旧版本，以防止文档丢失。

## 4. 典型查询示例
- 用户：“帮我把这个 `UserService.java` 文件里的所有注释提出来写成Markdown文档。”
- 用户：“读取 `src/main/java/com/example/` 目录，导出所有接口注释到API.md。”
